# Garbage Collection Cheat Sheet

## Core Rules

- Understand runtime behavior before syntax shortcuts.
- Keep transformations explicit.
- Validate assumptions with tests.

## Do and Avoid

Do:
- Predict before execute.
- Keep refactors incremental.
- Document one key insight after practice.

Avoid:
- Large mixed refactors.
- Hidden mutable state.
- Skipping edge case validation.

## One-Line Rule

If you can explain the runtime behavior clearly, your implementation is usually easier to trust.
