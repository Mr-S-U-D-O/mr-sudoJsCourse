# Portfolio Projects Track

This folder contains real-world, portfolio-oriented JavaScript projects.

These projects are intentionally open-ended. They are designed to train problem-solving, architecture thinking, and production habits.

## How To Use This Track

1. Pick one project.
2. Read that project's README.md to understand scope and motivation.
3. Read that project's guide.md and follow the implementation sequence.
4. Build your own solution in src/.
5. Use docs/ to record architecture, tradeoffs, and decisions.
6. Compare only after your attempt using solution/index.solution.js.

## Note On data/.gitkeep

Git does not track empty folders. A .gitkeep file is a tiny placeholder so empty data folders remain in version control.

You can replace .gitkeep with real sample data files later.
