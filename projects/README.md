# Portfolio Projects Track

These are full-blown JavaScript projects, not quick tasks.

Each project is designed to do two things at once:
- push students to figure out implementation details independently;
- produce a real, presentable portfolio artifact.

## How This Folder Works

1. Pick one project folder.
2. Read its README end-to-end before writing code.
3. Build from the provided scaffold in `src/` and `tests/`.
4. Use the step-by-step guide to decide what to build next.
5. Keep your own implementation in starter files.
6. Keep complete reference code only in `solution/`.

## Project Catalog

1. `01-calculator-engine`: expression parser + evaluation engine.
2. `02-chess-rules-engine`: legal-move validator and game-state engine.
3. `03-inventory-management-system`: stock, reorder, purchase, and audit logic.
4. `04-ticket-management-system`: ticket lifecycle, assignments, SLAs, and metrics.
5. `05-task-planner-api`: production-style REST API with persistence and auth.
6. `06-agent-workflow-orchestrator`: multi-step JavaScript agent orchestration.

## Completion Standard (Portfolio Ready)

A project is considered complete only when it includes:
- a working implementation;
- meaningful test coverage;
- architecture notes in `docs/`;
- clear run instructions;
- a polished README that explains tradeoffs and outcomes.

## Rules

- Do not copy-paste final answers from `solution/` during first attempt.
- Keep commits small and descriptive.
- Write tests before or alongside implementation where possible.
- Prefer readable, maintainable code over clever shortcuts.
