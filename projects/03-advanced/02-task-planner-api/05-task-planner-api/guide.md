# Guide: Task Planner API

## Build Order

1. Define data models: user, project, task.
2. Build registration/login and token verification.
3. Build project CRUD with ownership checks.
4. Build task CRUD and query filters.
5. Add pagination and sorting.
6. Add activity logging and response normalization.

## What To Search

- node js jwt auth flow
- express service layer pattern
- input validation middleware javascript
- api pagination strategies

## How To Think

- API is contract-first: define shape before implementation.
- Separate transport layer from business layer.
- Treat authz bugs as critical.
