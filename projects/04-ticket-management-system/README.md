# Project 04: Ticket Management System

## What This Project Is

You are building a ticket workflow engine for issue intake, assignment, SLA tracking, and operational metrics.

## Why This Project

This maps directly to support tools, helpdesk software, internal ops systems, and incident workflows.

## What You Should Know First

- object modeling and state transitions
- dates and time calculations in JavaScript
- defensive validation patterns
- basic metrics aggregation

## Rules

- enforce explicit status transition rules;
- track every important event in a timeline;
- compute SLA state from deterministic logic;
- keep business rules separate from presentation.

## Motivation

This project demonstrates backend product-thinking and workflow design ability.
