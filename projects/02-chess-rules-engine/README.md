# Project 02: Chess Rules Engine

## What This Project Is

You are building a chess game-state engine that validates legal moves and tracks result state.

## Why This Project

This project exercises deterministic rules, state transitions, and defensive coding under many edge conditions.

## What You Should Know First

- 2D board representations
- Objects, arrays, and loops
- Pure function design
- Basic chess rules

## Rules

- Keep board state immutable from the caller perspective.
- Reject illegal moves with clear errors.
- Separate move generation from move validation.
- Track turn and game status on each move.

## Motivation

Interviewers value this because it shows ability to model complex logic and reason about constraints.
