# Learning System

This course is designed as a progression system, not a random set of exercises.

## Daily Loop (60-90 minutes)

1. Learn: read one lesson README and summarize it in one sentence.
2. Build: implement the required function.
3. Test: run focused tests until green.
4. Break: intentionally test one wrong approach.
5. Explain: write a two-sentence reflection.

## Weekly Loop

1. Complete 4 to 6 lessons.
2. Re-run one old lesson from memory.
3. Complete one mini-project or checkpoint task.
4. Fill the weekly review template.

## Difficulty and Mastery

- Bronze: basic correctness and syntax use.
- Silver: edge-case handling and robust logic.
- Gold: trap resistance, clarity, and reasoning.

A learner moves forward only when they can:

- pass all required tests,
- explain one tradeoff,
- describe one bug they prevented.

## Stuck Protocol

If blocked for more than 20 minutes:

1. Re-read the lesson outcome and constraints.
2. Write failing test expectations in plain English.
3. Reduce the problem to one tiny example.
4. Compare expected vs actual output.
5. Ask: "What assumption am I making that may be false?"

## Fast Track and Recovery Track

- Fast track: if 2 lessons in a row feel easy, complete one stretch task.
- Recovery track: if stuck repeatedly, halve scope and solve the Bronze requirement first.

## Checkpoint Standard

At the end of each track:

- pass the capstone tests,
- complete one reflection file,
- explain the most important concept in plain English to a beginner.
