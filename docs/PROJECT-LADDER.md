# Project Ladder

This ladder turns topic mastery into portfolio-ready outcomes.

## Principles

- Projects are required, not optional.
- Scope increases by one dimension at a time: logic, state, async, architecture, performance.
- Every project must have acceptance tests.

## Tier 1: Foundations Builder

Target tracks: 01-Easy-Fundamentals, 02-Intermediate-Core

Project ideas:

- Grade analyzer CLI
- Expense categorizer
- String and data cleanup toolkit

Required outcomes:

- Clean function decomposition
- Input validation
- Readable test names

## Tier 2: Async and System Thinking

Target tracks: 03-Hard-Architecture, 04-Advanced-Deep-JS

Project ideas:

- Task queue simulator
- Event loop visual logger
- Promise orchestration utility

Required outcomes:

- Async failure handling strategy
- Concurrency reasoning
- Debugging notes in README

## Tier 3: Modern and Platform Mastery

Target tracks: 05-Modern-ECMAScript, 06-The-Platform

Project ideas:

- Streaming transform pipeline
- Worker-thread data processor
- Typed array binary parser

Required outcomes:

- Performance baseline and comparison
- Resource management notes
- Platform tradeoff analysis

## Tier 4: Architecture and Internals

Target tracks: 07-Functional-Architecture, 08-Engine-Internals-and-Dark-Arts

Project ideas:

- Functional rules engine
- Secure object merge library
- Runtime behavior profiler

Required outcomes:

- Explicit architecture decision record
- Security checklist
- Complexity and memory discussion

## Project Submission Standard

Each project folder should include:

- README.md with mission, constraints, and acceptance criteria
- implementation file(s)
- tests covering baseline, edge, and trap behavior
- reflection.md with:
  - design tradeoffs
  - bugs found and fixed
  - future improvements
