# Iterator Helpers Real-World Patterns

## Pattern 1: Explain-Then-Implement

1. State the rule.
2. Write smallest working code.
3. Add one edge case.

## Pattern 2: Const-First Declarations

- Start with const.
- Switch to let only when reassignment is necessary.

## Pattern 3: Small Safe Refactors

- Change one concept at a time.
- Run checks after each step.
